## Findings considered and rejected / deferred (so nobody re-audits them)

### Round 2 (2026-07-14, at `c874e13`)

- **REJECTED — "scheme handler does synchronous disk reads on the main actor"**: a perf subagent
  claimed `handleRequest`'s `Data(contentsOf:)`/`String(contentsOf:)` calls run on the main actor
  because they're awaited from a `@MainActor` Task. Wrong under this project's settings
  (`SWIFT_VERSION 6.0`, no `NonisolatedNonsendingByDefault`/approachable-concurrency flags):
  a nonisolated `static async` function hops to the global concurrent executor when awaited, so
  the reads already run off-main (`WebPageExtractor.swift:759,782`). No action. (If those flags
  are ever enabled, re-evaluate.)
- **No CSP on synthesized proxy responses + `Access-Control-Allow-Origin: *` on the asset route**
  (`WebPageExtractor.swift:671-681,915`) — confirmed HIGH-confidence update of round 1's deferred
  CSP finding: under the truthful-proxy design the handler is now the origin server and forwards
  none of the site's protections while injecting the privileged `vellum` bridge into every page.
  Still deferred for the same reason as round 1: a restrictive CSP can break SPA hydration and the
  injected script on strict sites; needs a design decision (nonce strategy, per-origin asset ACAO)
  plus manual QA budget. **Recommended as the first item of the next round.** Plan 007 meanwhile
  removes the worst capability reachable through the bridge.
- **Scheme-handler serving path has zero test coverage** (`handleRequest` 204-guard,
  `serveArchiveAsset` traversal guards, snapshot fallback — `WebPageExtractor.swift:782-943`) —
  real gap, M effort (needs fixture-backed `WebLibrary` root and a `WebFetch` seam). Below this
  round's cut line; good next-round plan after 001/010 settle.
- **Verified NOT findings this round** (so nobody re-reports): the model-selector crash fix is
  correct and complete (no stock `SecureField` remains; both reveal paths resign first responder);
  `AiToolEngine.displayActions` can't leak across turns (fresh engine per `sendMessage`);
  archive-asset route rejects path traversal (hex-only keys, `..`/`/`/`\` checks);
  `RevealableSecureField` doesn't log/paste/autofill key material; AI output rendering has no
  HTML/JS sink (AttributedString inline-only + SwiftMath images); notes and AI chat share one
  markdown pipeline (no duplicate renderer); no dead pre-proxy URL paths left in
  `WebViewerView.swift`.
