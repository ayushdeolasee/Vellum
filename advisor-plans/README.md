## Findings considered and rejected / deferred (so nobody re-audits them)

Vetted findings that did NOT become plans this round, with reasons:

- **App Sandbox / Hardened Runtime / code signing all disabled** (`project.yml:12-13,51-53`) — real
  and important for a distributed app, but L-effort with HIGH fix-risk (every file-access path needs
  security-scoped bookmarks) and it's a product decision about distribution posture. Needs a
  maintainer call before planning. HIGH confidence finding.
- **Web view strips page CSP + ATS `NSAllowsArbitraryLoads`** (`WebPageExtractor.swift:584-617`,
  `Info.plist:35`) — confirmed, but the fix (app-supplied CSP header on scheme-handler responses)
  risks breaking highlight-overlay injection on strict-CSP sites; needs a design decision + manual
  QA budget. Recommend it as the first security plan of a future round.
- **Page URL interpolated into inline `<script>` with `.withoutEscapingSlashes`**
  (`WebPageExtractor.swift:599,617`) — hardening gap, MED confidence on exploitability; fold into
  the CSP work above rather than a standalone plan.
- **Auto-archive fetches page-named subresources with no loopback/private-IP block** (blind SSRF,
  `WebArchive.swift:322-333`, `WebPageExtractor.swift:425-439`) — S–M fix (reject
  loopback/RFC1918/link-local before `fetchAsset`); worth doing, just below this round's cut line.
- **Prompt-injection surface of the tool-calling agent** — audited; containment verified
  (tools touch only the current document's annotations/navigation, write-capped at 5,
  `AiToolEngine.swift:25,51-53`). No action needed beyond keeping that containment.
- **Five provider clients duplicate ~150–200 lines each of streaming/retry/tool-schema plumbing
  with live drift** — the drift's user-visible bug is fixed by plan 003; the full extraction of a
  shared streaming layer is L-effort/MED-risk and should happen deliberately, after 001+003 settle.
- **`WebContentScript.swift` is an 1800-line JS string with no lint/tests** — its own header says
  it's a verbatim port that "must not drift" (anchor compatibility), so treating it as
  intentionally frozen for now; revisit when the first real JS change is needed (move to a bundled
  `.js` resource + JSCore tests for the pure functions then).
- **`AnnotationStore` failures are silent** (NSLog-only revert, no `error` property unlike
  AppStore/AiStore — `AnnotationStore.swift:78,158,185`) — good S-size candidate for the next
  round; needs a small UI decision (where the banner lives).
- **`annotationsForPage` is a linear scan re-run per visible page on every scroll/zoom tick**
  (`AnnotationStore.swift:202`, `PdfOverlays.swift:101-109`) — MED confidence on real-world impact;
  profile with an annotation-heavy document before building the page index.
- **`SWIFT_STRICT_CONCURRENCY: minimal` on a Swift 6 project** with several
  `nonisolated(unsafe)`/`@unchecked Sendable` escape hatches — worthwhile ratchet
  (minimal → targeted → complete) but M-effort with real diagnostics to resolve; schedule
  deliberately.
- **macOS 26.0 minimum deployment target** — README drift is fixed by plan 006; whether the floor
  itself should drop (it enables Liquid Glass APIs but excludes most installed Macs) is a product
  decision, not a plan.
- **No CI** — high leverage but blocked on 001 (the first run would be red) and on a maintainer
  decision about macOS runner cost. One workflow file:
  `xcodegen generate && xcodebuild -project Vellum.xcodeproj -scheme Vellum -destination 'platform=macOS' test`.
- **No SwiftLint/formatter** — fine S follow-up once CI exists to enforce it.
- **Trivia not worth plans** (one-line fixes, fold into any passing commit): redundant double
  session-guard in `AnnotationStore.updateAnnotation`/`deleteAnnotation` (`AnnotationStore.swift:132-136,167-168`);
  stale "6 iterations" comment in `GeminiClient.swift:12` (fixed as part of plan 001).
- **FIXLIST.md items** — all verified already fixed at `314cf9f`; plan 006 banners the file so
